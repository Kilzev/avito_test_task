

## Требования

- Python 3.10+
- pip

## Быстрый старт (venv)

```bash
python -m venv .venv
.venv\Scripts\activate
python -m pip install --upgrade pip
pip install -r requirements.txt
```

## Установка зависимостей

```bash
pip install -r requirements.txt
```

## Запуск тестов

```bash
pytest -v
```

Переменные окружения:
- `API_BASE_URL` — базовый URL сервиса (по умолчанию `https://qa-internship.avito.com`).

## Что покрыто
Тесты в `tests/test_api.py` покрывают кейсы из `TESTCASES.md`:
- Создание объявления (валидные и невалидные сценарии, в том числе дубликат sellerId и некорректные типы name/sellerId/price).
- Получение объявления по id (валидный, несуществующий, невалидный id).
- Получение объявлений по sellerId (валидный, пустой результат, невалидный sellerId).
- Получение статистики (валидный, несуществующий, невалидный id).
- Удаление объявления (валидный id, несуществующий id, невалидный id).

## Важные нюансы
- Для уникальности `sellerId` тесты используют случайное значение в диапазоне 111111–999999.
- После создания объявления id извлекается через `GET /api/1/{sellerId}/item`, поэтому наличие в ответе поля `name` критично.
- Тест `test_tk14_get_items_by_seller_empty` допускает, что API может вернуть 404 вместо пустого массива; в этом случае тест помечается как skipped.
